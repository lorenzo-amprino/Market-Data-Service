# Use local read APIs with asynchronous ingestion

Market Data Service canonical read APIs will serve consumers from the local database and will not call external Data Providers synchronously during ordinary search, lookup, detail, price, dividend, or corporate-action requests. Explicit discovery search endpoints may call Data Providers synchronously to return best-effort candidate instruments or listings that are not yet known locally, but those candidates are not canonical service data until an acquisition workflow creates them and ingestion/backfill stores their data locally. This protects normal internal consumers from provider latency, outages, rate limits, and license constraints, while still allowing interactive discovery for personal-use onboarding; freshness of canonical data is exposed through Data Availability, Data Provenance, and retrieval timestamps, and ingestion jobs are responsible for keeping the local store current.
